    Otel configuration¶

    Splunk OpenTelemetry Collector is a component that provides an option to send metrics to signalFx. In order to use it, you must set enabled flag in values.yaml to true:

    sim:
  # sim must be enabled if you want to use signalFx
  enabled: true

    Also you need to specify signalFx token and realm, so at the end sim element in values.yaml looks like this:

    sim:
  enabled: true
  signalfxToken: BCwaJ_Ands4Xh7Nrg
  signalfxRealm: us0

    After executing microk8s helm3 upgrade --install snmp -f values.yaml splunk-connect-for-snmp/splunk-connect-for-snmp --namespace=sc4snmp --create-namespace, the sim pod should be up and running:
